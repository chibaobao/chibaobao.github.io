vlc 框架概述

vlc功能：

  • 服务器
    • 直播服务器
    • 点播服务器
  • 客户端

vlc中的一些概念

  • input:输入可以是文件或者流
  • access： access表示I/O访问，打开文件/流提供read,seek,peek当方法供stream使用
  • stream: stream表示流，将不同的access统一抽象为stream类型（通过回调调用access提供的方法），dumux层直接调用stream
  • demux: demux表示解复用，从stream中读取数据，通过es_out发给decoder
  • es_out: (通过fifo)该层获取demux后的数据，送decode解码。
  • decode: 表示解码，解码后送output。
  • output: 表示输出，获取从decode出来的数据，送readerer或者通过网络发出。
  • readerer: readerer表示显示
  • fifo:VLC封装的一个队列（类似于管道），进程间数据传输使用（通过block_FifoGet，和block_FifoPut函数）

vlc的工作流程
input–>access–>stream–>demux–>es_out–>decode–>output

vlm使用及源码分析

使用

vlm使用官方文档

  • 启动：vlc --ttl 12 -vvv --color -I telnet --telnet-password videolan --rtsp-host 0.0.0.0 --rtsp-port 554
  • 控制端连接 telnet 127.0.0.1 4212
  • 用telnet执行以下命令：
    • new Test vod enabled 创建一个点播服务器
    • setup Test input my_video.mpg 为点播服务器添加输入流
  • 客户端：vlc rtsp://server:554/Test进行播放

源码（有用断点处）

vlc部分模块分析

rtsp模块

  • 文件modules/streamout/rtsp.c和rtp.c,rtcp.c,rtpfmt.c,vod.c
  • open()位于rtp.c中，如果是要调rtsp功能则在open中创建一个httpserver,rtp.c启动一个打包线程将数据打成rtp的包，一个发送线程负责发送数据
  • 在rtsp中rtsp_callback函数负责处理http请求根据请求控制rtp发包情况,get请求参数可以通过(char*)(query->psz_args)获取
